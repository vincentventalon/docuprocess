# SaaS Pricing Models Reference

## Common Pricing Models

### Subscription (Per Seat/User)
- **How it works**: Fixed price per user per month/year
- **Best for**: Collaboration tools, team software
- **Examples**: Slack, Notion, Linear
- **Typical pattern**:
  - Free tier (limited users/features)
  - Pro: $10-25/user/mo
  - Business: $25-50/user/mo
  - Enterprise: Custom

### Usage-Based (Pay-as-you-go)
- **How it works**: Pay for what you use (API calls, storage, etc.)
- **Best for**: Infrastructure, APIs, developer tools
- **Examples**: AWS, Twilio, Stripe
- **Typical pattern**:
  - Free tier with limits
  - Pay per unit after threshold
  - Volume discounts at scale

### Tiered/Feature-Based
- **How it works**: Different feature sets at different prices
- **Best for**: Products with clear feature progression
- **Examples**: Mailchimp, HubSpot, Ahrefs
- **Typical pattern**:
  - Free/Starter: Core features
  - Pro: Advanced features
  - Enterprise: Everything + support

### Flat-Rate
- **How it works**: One price for everything
- **Best for**: Simple products, specific use cases
- **Examples**: Basecamp
- **Typical pattern**: Single price, unlimited users

### Hybrid Models
- **How it works**: Combination of above
- **Example**: Base subscription + usage overage
- **Common in**: Email tools, analytics platforms

## Pricing Tiers (Typical Structure)

### Free Tier
- **Purpose**: Lead generation, product-led growth
- **Limits**: Features, usage, users, or time
- **Types**:
  - Freemium (forever free, limited)
  - Free trial (time-limited, full access)
  - Developer tier (free for low usage)

### Starter/Basic ($10-50/mo)
- **Target**: Individuals, small teams
- **Typical limits**:
  - 1-5 users
  - Basic features
  - Limited support

### Pro/Professional ($50-200/mo)
- **Target**: Growing teams, SMBs
- **Typical includes**:
  - 10-25 users
  - Advanced features
  - Priority support
  - Integrations

### Business/Team ($200-500/mo)
- **Target**: Mid-market companies
- **Typical includes**:
  - Unlimited or high user limits
  - Admin controls
  - SSO (sometimes)
  - API access

### Enterprise (Custom pricing)
- **Target**: Large organizations
- **Typical includes**:
  - Everything in lower tiers
  - SSO/SAML
  - Custom contracts
  - Dedicated support
  - SLAs
  - On-premise options

## Pricing Strategies

### Value Metric Selection
- **Per user**: Easy to understand, scales with adoption
- **Per resource**: Storage, contacts, API calls
- **Per feature**: Feature-based tiers
- **Per outcome**: Transactions, leads generated

### Anchor Pricing
- Show expensive option first to make others seem reasonable
- "Most popular" badge on middle tier

### Annual Discount
- Typical: 15-20% discount for annual billing
- Purpose: Improve cash flow, reduce churn

### Localized Pricing
- Different prices for different regions
- Purchasing power parity (PPP)

## Pricing Page Patterns

### Standard Elements
- 3-4 pricing tiers (2-3 is also common)
- Feature comparison table
- "Most popular" highlight
- Annual/monthly toggle
- FAQ section
- Social proof (logos, testimonials)

### Common CTAs
- "Start free trial"
- "Get started"
- "Talk to sales" (enterprise)
- "Book a demo"

### Trust Elements
- Money-back guarantee
- No credit card required
- Security badges
- Customer logos

## Analysis Framework

When analyzing competitor pricing:

1. **Model**: What pricing model do they use?
2. **Metric**: What's the value metric?
3. **Tiers**: How many tiers? What's in each?
4. **Entry point**: What's the cheapest paid option?
5. **Enterprise**: Do they have enterprise/custom pricing?
6. **Free tier**: Freemium or trial?
7. **Billing**: Monthly/annual options?
8. **Discounts**: What discounts are visible?

---

## Red Flags

- No public pricing (usually means enterprise-only or very expensive)
- Confusing pricing page (may indicate frequent changes)
- Many add-ons (true cost is higher)
- Per-seat AND usage pricing (costs scale fast)
- Missing feature comparison (hiding limitations)

---

## Market Position by Pricing

- **Premium**: 2x+ market average (positioning on quality/features)
- **Mid-market**: Market average (balanced value)
- **Budget**: 50% of market average (competing on price)
- **Free/Open-source**: $0 (monetize differently)
